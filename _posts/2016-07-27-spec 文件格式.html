spec 文件格式

文件头

Name：名字
Summary：摘要
Version：软件版本号
Release：软件包释出号
Vendor：公司/开发者
License：版权 ( GPL )
Group：软件包所属类别enter link description here
Source：源包名字（ demo.tar.gz ）
Requires: 该rpm包所依赖的软件包名称，可以用>=或<=表示大于或小于某一特定版本。还有例如PreReq、Requires(pre)、Requires(post)、Requires(preun)、Requires(postun)、BuildRequires等都是针对不同阶段的依赖指定

%prep 预处理阶段

  • %setup -n %{name}-%{version} 把源码包按指定名字解压并放好，通常用%setup -c
  • %setup 不加任何选项，仅将软件包打开。
  • %setup -n newdir 将软件包解压在newdir目录。
  • %setup -c 解压缩之前先产生目录。
  • %setup -b num 将第num个source文件解压缩。
  • %setup -T 不使用default的解压缩操作。
  • %setup -T -b 0 将第0个源代码文件解压缩。
  • %setup -c -n newdir 指定目录名称newdir，并在此目录产生rpm套件。
  • %patch 最简单的补丁方式，自动指定patch level。
  • %patch 0 使用第0个补丁文件，相当于%patch ?p 0。
  • %patch -s 不显示打补丁时的信息。
  • %patch -T 将所有打补丁时产生的输出文件删除。

%build 编译阶段

  • %configure ：rpm定义的标准宏命令，执行源代码的configure配置
  • make
  • 实际编译方法根据自己项目编写对应shell命令

%install：安装阶段

  • make DESTDIR=$RPM_BUILD_ROOT install

  • 或者使用 install -d $RPM_BUILD_ROOT/

    install -m 755 helloworld /usr/local/bin/ 以指定文件属性安装到指定目录

  • 或者使用 cp -a * $RPM_BUILD_ROOT/

%files：定义哪些文件或目录以何种属性放入rpm中

%defattr (-,root,root) 指定包装文件的属性，分别是(mode,owner,group)，-表示默认值，对文本文件是0644，可执行文件是0755
%config(noreplace) %{installdir}/conf/ns.conf
%doc $RPM_BUILD_DIR/demo/doc/README.md

%clean 清理临时文件

%pre rpm安装前执行的脚本

%post rpm安装后执行的脚本

%preun rpm卸载前执行的脚本

%postun rpm卸载后执行的脚本

%changelog 变更日志
